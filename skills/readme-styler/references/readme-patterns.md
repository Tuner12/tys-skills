# README Patterns

Use these patterns to choose the right README shape.

## 1. Polished Public Repo

Best for:

- personal projects
- public tools
- repository showcases

Suggested sections:

- centered hero
- one-line value statement
- badge row
- overview
- highlights
- repository layout
- usage or examples
- roadmap

## 2. Utility or Library Repo

Best for:

- developer tools
- packages
- reusable libraries

Suggested sections:

- title and summary
- install
- quick start
- API or examples
- configuration
- contributing

## 3. Collection Repo

Best for:

- skill libraries
- template collections
- curated resources

Suggested sections:

- hero
- overview
- featured items
- table of contents or inventory table
- repository structure
- how to add more items

## 4. README Upgrade Heuristics

When rewriting a weak README:

- reduce long intro paragraphs
- make the first screen more informative
- turn vague prose into sections with purpose
- add a feature list or table if it improves scanning
- add only the badges that communicate something useful
- keep wording aligned with the actual project quality

## 5. Visual Tricks That Usually Work

- centered hero for public-facing repos
- one badge row in a single visual style
- one callout note near the top
- one feature table or inventory table
- one repository tree block
- a restrained footer line

## 6. Visual Tricks That Often Fail

- mixed badge styles
- too many shields with low information value
- giant animated GIFs for technical repos
- decorative emojis in every heading
- banners that overpower the actual content
